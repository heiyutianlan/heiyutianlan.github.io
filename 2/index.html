一些自己的调整

注: 在网页中可以使用f12来查看每个元素，同时进行调整，直到自己满意再修改文件

图标修改

  • 参考这里找到了图标的一个总文件 blong\themes\matery\source\libs\awesome\css 摘录如下

    /*!

    • Font Awesome Free 5.11.1 by @fontawesome - https://fontawesome.com
    • License - https://fontawesome.com/license/free (Icons: CC BY 4.0, Fonts: SIL OFL 1.1, Code: MIT License)
    • /

    也同样认证了我的判断
    我之前使用的图标是https://fontawesome.dashgame.com/#basic
    但他的版本与matery主题使用的版本不一样，所以名称与使用方式有改变，自然无法显示

网站logo与网站名

  • 网站名称为了手机的显示调整了大小，使得logo与名称不在中线
    f12找到 nav .brand-logo 中 font size 可以调整显示大小 右方有其文件materialize.main.css :1(表示行数)
    调整 font size 的参数直到自己满意
    在 blong\themes\matery\source\libs\materialize 找到文件 materialize.main.css 第一行太长，用查找找到 nav .brand-logo 中 font size 修改就好
  • 修改介绍页图标
    介绍页图标是圆的，使得图标有一部分被遮挡，使用f12查看元素，在style中，我实验了各种元素对他的影响，发现circle的参数可以将图标改成全显，其之后的文件显示是在 materiaize.min.css 的第一行，找到circle将他的参数改为0%就好。需要学习啊
  • 修改导航页的副标题
    在 blong\themes\matery\languages\zh-CN.yml 对翻译进行修改
    如果没有 f12 查找元素
  • 修改导航页的图标
    查找元素没有找到，突然想到 blong\themes\matery\layout 中各个导航页的ejs文件
    比如 categories: 分类（分类是我改的） 页
    找到 layout 下 categories.ejs 打开有一句

    <%- partial(‘_widget/category-cloud’) %>

    一开始不明白，之后知道了_widget/category-cloud是文件位置
    打开位于_widget文件夹中的category-cloud文件，里面 “fab fa-wpbeginner”（这是我试验时改的与原文件不一样）改成 https://fontawesome.com/ 网站中你想使用的icon名称就可以了。
    到这一步，副标题的图标就改好了。

写博客

markdown

空格敏感
空行另一行，要不然会当成相同的一行进行处理，同时首行是什么样子，后面的即使是换行也会以首行的方式进行处理。
文字为起始以空格数量为标记，注意边缘。

图片插入

  • 在文章的Front-matter中的img中插图片的经历
    在文章的Front-matter中的img中插图片
    一开始是直接看的https://hexo.io/zh-cn/docs/asset-folders
    用 hexo g 报错
    很疑惑换了路径blong\source_posts\下面的博文同名的资料文件夹内的图片不显示
    从用法错误到怀疑是无法识别路径因为“_”标记的文件会忽略下载了一些插件到最后看到了这个标签插件第一句“标签插件和 Front-matter 中的标签不同，它们是用于在文章中快速插入特定内容的插件。”好吧，然后看https://blinkfox.github.io/2018/09/28/qian-duan/hexo-bo-ke-zhu-ti-zhi-hexo-theme-matery-de-jie-shao/#toc-heading-19的示例 /source/images/xxx.jpg，所以我需要一个图片文件夹来存放图片，那我直接创建一个图片页来存放我的图片，也方便我引用。
    在文章的Front-matter的img与coverImg中都是用images/xxx.jpg来指定图片路径搞定。
