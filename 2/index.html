一些经历

  1. 注意
    设置 npm 的内置路径 ——> 全局模块路径和缓存路径
    在这一步修改了全局命令文件位置有问题 之后要详细看看

  2. 上传之前
    在博客文件夹内执行

    安装部署插件

  3. 生成ssh密钥
    GitHub文档

    可参考
    ookamiAntD
    第10条 配置SSH与Git

  4. 查看主分支名称
    在博客站点配置文件config.yml
    修改 branch 默认master 改成 yourname.github.io 仓库的主分支名称

  5. 模块找不到
    参考
    internal-modules-cjs-loader-js883-throw-err
    多次出现包括生成与部署时，注意不要随便 hexo clean

  6. 换行符的问题
    LF will be replaced by CRLF
    在部署时出现，可以不理会。
    参考网址

    • 退休码农飞伯德(注意这篇文章的许可协议)
    • how-to-force-consistent-line-endings-in-git-commits-with-cross-platform-compatib
    • make-git-lf-will-be-replaced-by-crlf-warnings-go-away
    • lf-will-be-replaced-by-crlf-in-git-what-is-that-and-is-it-important

  7. 站点配置文件 config.yml

    • URL配置
      要加http或https
      刚开始没注意报错 FATAL TypeError [ERR_INVALID_URL]:
    • home page setting
      path 根路径设置为默认值’’,修改为其他值有时在本地可以打开网址，但部署之后显示404

  8. 主题配置文件 config.yml

    url也不需要配置。配置之后首页标签会出问题。或者 配置错误

一些自己的调整

  • 在网页中可以使用f12来查看每个元素，同时进行调整，直到自己满意再修改文件

    • 图标修改

      • 参考这里找到了图标的一个总文件 blong\themes\matery\source\libs\awesome\css 摘录如下

        /*!

        • Font Awesome Free 5.11.1 by @fontawesome - https://fontawesome.com
        • License - https://fontawesome.com/license/free (Icons: CC BY 4.0, Fonts: SIL OFL 1.1, Code: MIT License)
        • /

        也同样认证了我的判断
        我之前使用的图标是https://fontawesome.dashgame.com/#basic
        但他的版本与matery主题使用的版本不一样，所以名称与使用方式有改变，自然无法显示

    • 网站logo与网站名

      • 网站名称为了手机的显示调整了大小，使得logo与名称不在中线
        f12找到 nav .brand-logo 中 font size 可以调整显示大小 右方有其文件materialize.main.css :1(表示行数)
        调整 font size 的参数直到自己满意
        在 blong\themes\matery\source\libs\materialize 找到文件 materialize.main.css 第一行太长，用查找找到 nav .brand-logo 中 font size 修改就好

    • 修改介绍页图标

      • 介绍页图标是圆的，使得图标有一部分被遮挡，使用f12查看元素，在style中，我实验了各种元素对他的影响，发现circle的参数可以将图标改成全显，其之后的文件显示是在 materiaize.min.css 的第一行，找到circle将他的参数改为0%就好。需要学习啊
